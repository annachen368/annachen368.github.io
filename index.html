Improved performance for contact loading and service calls when handling 2000+ contacts

Problem:

  • The previous implementation checked the enrollment status of contacts in batches of 200, but the service calls ran on the main thread, causing UI lag during scrolling.
  • To avoid lag, the logic was changed to trigger all service calls at once on launch, which made the screen load extremely slowly for users with large contact lists.

My Solution:

  • Implemented pagination to load and check enroll status for 200 contacts at a time as the user scrolls.
  • Used background coroutines to move service calls off the main thread.
  • Added a 300ms debounced search and suppressed repeated queries to reduce unnecessary requests.
  • Introduced a Mutex to prevent duplicate concurrent status checks.
